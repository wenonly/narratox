> **一句话**：把散落的 7 份大纲模板/写法文件，合并成一份「从要不要写、到怎么写、到直接套用模板」的大纲写作终极指南。
>
> **写作价值 / 适用场景**：这是写长篇/网文前**最先该看的一份**。大纲是故事的骨架——骨架立不住，正文写得再花也会越写越散。本篇解决三个问题：①要不要写大纲；②大纲怎么写（粗纲/细纲/给编辑/给自己）；③直接给可套用的模板。

---

## 一、大纲是什么 / 要不要写

**大纲 = 整个小说的纲要，是写作前对故事的整体构造和把握。**

- **粗纲**：一句话描述整个事件。例：「莫北黑了罗凯泽的账号，匿名威胁他离开俞秀筠。」
- **细纲**：把过程情节做大致描述（粗纲是树干，细纲是枝杈，正文是树叶）。

**一定要写吗？因人而异：**
- **即兴创作者**：只有一个创意+一个主角就开写，随主角冒险深入而生长——可以不写细纲，遵循自己的习惯。
- **非即兴创作者（大多数人/长篇）**：**必须写**。不先拟大纲，长篇会越写越散、越写越乱、容易卡文。先把「什么环境逆袭、设置哪些人物、谁帮谁阻、主角怎么成长」想清楚，才能下笔。

> 🧠 新人常卡在大纲上磨灭热情。解法：**不必一步到位**——先写粗纲（一句话主线），边写边补细纲。大纲是活的，可以随写随改。

## 二、给编辑的大纲 vs 给自己的大纲

| | 给编辑的（投稿用） | 给自己的（写作用） |
|---|---|---|
| **目的** | 让编辑快速判断「写什么、有没有卖点」 | 把控全局，防止写散 |
| **长度** | 一两千字足够，超长篇上限最好不超 6000 字 | 越详尽越好（一二百万字的长篇，提纲至少 8000–10000 字） |
| **核心三要素** | ①主人公是谁 ②主要做什么事 ③结局如何 | 主线+辅线+人物+设定+分章细纲，全部要细 |
| **注意** | 编辑一天看太多稿，必须让他快速决定签不签 | 还可私下写「分集介绍」/给章节取概括性小标题，防止思路乱 |

## 三、细纲必含的内容（写法要点）

1. **题材**：明确写什么题材，免得后面写偏（都市写着写着变玄幻水字数）。
2. **小说特色、卖点**：凭什么让读者付费？新奇世界观？逆袭爽感？爱恨情仇？（提前想清楚）
3. **定位、读者群**：写给哪类读者，怎么写更能迎合他们。
4. **相关设定**：世界观、力量体系等。
5. **故事梗概（主线）**：至少 500 字，讲清「你要讲一个什么故事」。

## 四、章节细纲九要素（最实用的颗粒度）

写每一章的细纲时，按这 9 项填，情节自动立体（范例来自玄老怪）：

| 要素 | 内容（范例） |
|------|--------------|
| **渴望** | 保护老师俞秀筠 |
| **障碍** | 身份差距、内向性格、年龄差距 |
| **冲突** | 莫北 VS 罗凯泽 |
| **暗中** | 莫北通过网络进行 |
| **明面** | 莫北干扰罗凯泽的行动 |
| **行动** | 莫北黑了罗凯泽的账号，匿名威胁他离开俞秀筠，失败后发匿名信给俞秀筠曝光其所作所为 |
| **情感** | 正义感，喜欢御姐 |
| **展示** | 主角的身份、高超技术、喜欢御姐 |
| **爽点** | 黑了英语老师追求者的社交账户，发现这人人渣、一脚踏几条船、专门骗感情白痴的美女 |

> 🧠 这套「渴望→障碍→冲突→行动→爽点」是章节细纲的黄金结构——每一章都该有一个明确的渴望、一个阻碍它的障碍、一次冲突行动、一个爽点落点。照着填，章节不会水。

## 五、大纲模板（整合 5 份模板的最佳结构，可直接套用）

### 模板 A：通用网文大纲（推荐）

```
一、基础
  书名 / 预计字数 / 题材类型 / 基调（正剧·喜剧·悲剧·轻松·爆笑·暗黑）/ 文字风格

二、故事梗概（主线，≥500字）
  起因 / 发展 / 转折 / 高潮 / 结局

三、线网设定
  主线 / 辅线1 / 辅线2 / 事件线 / 感情线（高潮·起伏·低谷）/ 注意前后逻辑

四、世界观与设定
  环境地理（中央/东南西北）/ 势力分布 / 扩展设定（货币体系·政治体系·历法制度）/ 力量等级

五、人物设定（主角详细，配角可略）
  每人：姓名 / 年龄 / 性别 / 外貌（发色发型眼睛饰物）/ 性格 / 特征 / 习惯 / 背景 / 技能（武功或特长）/ 虐点·萌点

六、人物关系（用「身份-关系」链条写，参考仙剑）
  例：李逍遥——蜀山第27代掌门、赵灵儿之夫、李忆如之父、酒剑仙之徒……

七、细化情节（分章/分卷）
  卷一：简述 + 详情（什么背景下主角相遇 → 什么事件破坏关系 → 误会如何解决……）
  卷二：……

八、结尾设定（见 §六）

九、文案（简介，≤300字，要生动抓人，吸引读者）+ 关键字
```

### 模板 B：情节结构骨架（大纲完整模板·第一种模式的情节层）

把整本书的情节按成长阶段切片：

```
主角出世 → 生活环境 → 引发情节 → 主角入世
→ 学习 → 受挫 → 组队 → 敌对 → 战斗 → 进化 → 再学习 → 大势力 → 书末战斗 → 结文
敌对线：介绍 → 结仇 → 内部对话 → 分裂或发展
盟友线：介绍 → 结盟 → 帮助
```

### 模板 C：卷宗式（适合超长篇）

每卷一行：`卷名 / 简述 / 详情`，把百万字切成若干卷，每卷自成一段起承转合。

## 六、结尾设定的 5 种类型

| 类型 | 说明 |
|------|------|
| **死** | 主角死一个，另一个悲剧 |
| **大团圆** | 相爱的人在一起，磨难解除，全员幸福 |
| **留悬念** | 结局给悬念，让人猜不透主角未来 |
| **揭秘** | 秘密一个个揭开 |
| **……** | 无数种，写文起初就定好走向 |

## 七、八大要素（提纲速查）

故事背景 / 主要人物 / 世界设定 / 武功（能力）设定 / 单集详解 / 大纲总纲 / 特殊设定 / 注解。

## 八、多人协作 / 续稿要点

- 组队拼写流程：主题提出 → 情节规划 → 初建提纲 → 创意补充 → 情节补充 → 确立提纲 → 包干分配 → 回稿审核 → 章节修改 → 拼接润色 → 漏洞群修 → 发布。
- 风格与语言修改必须由同一人完成。
- 需他人续稿时，要提供详细提纲和思路，并保留续稿者个人标记（「续貂」：狗尾续貂，对原作者的尊重）。
- 留伏笔或新思路，在节末注明，利于后文。

### 🧠 整理思考

**结构判断**：这是公式模板里**最大的一次合并**——多份大纲写法/模板讲的都是「大纲怎么写」，切面不同（有的给模板、有的给写法、有的给范例），重叠度极高。整合成一份「为什么写→粗纲细纲→给编辑给自己→章节九要素→3 种模板→结尾 5 类→八大要素→协作」的完整指南，去重补全。

**知识提炼**：
1. **大纲 = 粗纲（一句话）+ 细纲（章节级）**，新人先写粗纲边写边补，别卡死。
2. **给编辑的（1–2 千字，讲清主人公/做什么/结局）≠ 给自己的（越细越好）**。
3. **章节细纲九要素（渴望/障碍/冲突/暗中/明面/行动/情感/展示/爽点）**是最实用的颗粒度——每章照填不水。
4. **结尾要在动笔前定好**（5 种类型），否则收不住。

**使用建议**：开新文时，照「模板 A」填一遍基础+梗概+人物，再用「章节九要素」逐章细化。卡文时回到大纲查「这章的渴望/障碍/爽点是什么」。投稿时把大纲压到 1–2 千字（主人公+主线+结局+卖点）。
