# 知识库整理规范（v2 · 精简）

> 本文件定义「整理版/」目录下每个 `.md` 的统一格式与整理动作。
> 目标：把 633 篇 OCR/抽取来源的原始条目，整理成**结构化、可被 AI 按需加载、可被人直接读**的高质量知识文档。

---

## 一、为什么要整理（现状诊断）

随机抽样 4 类文件后，发现原始条目普遍存在三类问题：

| 问题 | 表现 | 示例 |
|------|------|------|
| **OCR 错字泛滥** | 同音/形近字误识，影响理解 | 「展现截然不同的特征」→「展狐/蔭态」；「欧阳」→「欧那」 |
| **版式污染** | PDF/图片分栏、水印、页眉混入正文 | 「NS 风」「给0泡youyouyou 幕布」「8」「上」等孤立字符 |
| **结构坍塌** | 原本的「定义/特征/案例」分层，OCR 后挤成一团 | 傲娇毒舌：定义与特征、成长背景与案例互相穿插 |

**整理版要做的事**：重排结构、订正错字、补全语义，并加入「整理思考」让知识可推理、可复用。**不保留任何溯源/过程元数据**（source / 抽取方式 / OCR 质量 / 整理状态 / 字数等）——整理完它们即无意义。

---

## 二、统一格式

### 2.1 Frontmatter（极简，只留检索价值）

```yaml
---
tags: [人设]                # 内容分类标签，供 agent 按需加载时粗筛
---
```

就这一个字段。`适用场景`、`写作价值` 不放进 frontmatter，而是写进正文顶部的 blockquote（见 §2.2）——它们是「内容」，该和人一起读，不是机器字段。

### 2.2 正文（内容层）

正文用**分区 + 表格 + 引用**重建结构，规则：

1. **顶部「一句话 + 写作价值/适用场景」**：用 blockquote 浓缩这篇知识的核心与「为什么有用、什么场景用」。这是整篇的 TL;DR，供 agent 快速判断是否需要深读。
2. **分级标题重建结构**（`##` / `###`）：把 OCR 坍塌的内容按语义重新分层。
3. **能表格化的尽量表格化**：特征清单、案例对照、参数配置等，一律用表格（比散文更利于检索与对比）。
4. **订正 OCR 错字**：在保持原意前提下改对，不改写作者本意。
5. **删掉版式污染**：水印、分栏碎片、广告语、与正文无关的孤立字符直接删。
6. **保留原文金句**：台词、范例描写、桥段原文照录（这是知识库的核心资产，一字不改）。

### 2.3 正文末尾固定收尾：`### 🧠 整理思考`

每篇结尾必须有这一节，写**我（整理者）的判断与增量**：

- **结构判断**：这篇属于什么类型，我为什么这样分区。
- **知识提炼**：从原文抽出的、可迁移的方法论/规律（原文散落，我帮你收拢成一条）。
- **使用建议 / 注意事项**：怎么用、有什么坑、与哪类题材搭配。

> 「整理思考」是整理版相对原始条目的核心增量——它让死资料变成可推理的知识。后续 agent 即使只读这一节，也能拿到这篇的「应用指南」。

### 2.4 文档骨架清单（内部参考，不写入文件）

整理时按内容形态选骨架（仅作整理指引，不作为 frontmatter 字段）：

| 形态 | 典型来源 | 正文骨架 |
|------|----------|----------|
| **定义型** | 人设档案（反差萌/傲娇毒舌…） | 一句话 → 定义 → 特征表 → 行为准则 → 成长背景 → 案例表 → 描写范例 |
| **访谈型** | 人设档案（给 XX 做访谈） | 一句话 → 基本信息 → 性格内核 → 成长经历 → 关系网 → 台词样本 |
| **塑造法型** | 人设档案（三步塑造…）/ 创作须知 | 一句话 → 方法论总览 → 分步详解 → 要点清单 → 反例/坑 |
| **词汇型** | 词汇素材库 | 一句话 → 分类词表（表格） → 使用示范 |
| **公式型** | 公式模板 / 短篇写作公式 | 一句话 → 公式骨架 → 逐段拆解 → 范例套用 → 变体 |
| **拆文型** | 拆文案例 | 一句话 → 作品基本信息 → 优点提炼 → 结构拆解（起承转合） → 可复用点 |
| **方法论型** | 方法论教程 / 创作须知 | 一句话 → 核心观点 → 分点论述 → 操作清单 → 案例 |

不同形态用不同骨架，但都遵守 §2.2 的分区/表格/思考规则。

## 三、整理动作清单（每篇必做）

- [ ] 读原文，判定形态、选骨架
- [ ] 写 frontmatter（仅 `tags`）
- [ ] 写顶部「一句话 + 写作价值/适用场景」
- [ ] 按骨架重建正文结构
- [ ] 表格化清单/案例/对照
- [ ] 订正 OCR 错字（保持原意）
- [ ] 删除版式污染
- [ ] 写「🧠 整理思考」
- [ ] 校对：金句原文是否照录、有无改写作者本意

---

## 四、目录与命名

- 输出目录：`知识库/整理版/<分类>/<原文件名去掉序号>.md`
  - 例：`条目/人设档案/反差萌人设_0122.md` → `整理版/人设档案/反差萌人设.md`
  - 去掉 `_0122` 序号，让文件名更可读。
- 原始 `条目/` 目录**不动**（作为只读底本，可回溯、可重跑整理）。

---

## 五、质量底线

1. **不丢信息**：原文每个知识点都要在整理版里能找到对应（结构变了，内容不能少）。
2. **不改作者本意**：订正是「改对」，不是「改写」；拿不准的 OCR 字，宁可保留原文 + 「[原文如此]」标注，也不臆测。
3. **思考要有依据**：「整理思考」里的判断要能从原文找到支撑，不空谈。
