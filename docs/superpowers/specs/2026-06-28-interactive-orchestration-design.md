# 交互式编排设计(Phase 16)

> 日期:2026-06-28 · Phase 16 · 关联 [Phase 14](./2026-06-27-main-agent-reinforce-design.md)(主 agent 固化)

## 问题诊断

E2E 暴露:main agent **一口气自主跑完整条流水线**(一轮 curator+world+outline+character,15+ 分钟被超时掐)。立项时一股脑列 7 项 missing 让用户填。用户不在循环里——没法在每步介入、调整、给建议。

当前 MAIN_AGENT_PROMPT 虽有「等作者确认」字样,但实际行为是自主批处理(deepseek 倾向于一次做完所有事)。

## 目标

把 main agent 从**自主批处理**变成**交互式一步步协作**:每步做完停下,给建议 + 问用户「继续/调整/看看?」。用户随时在循环里。

## 设计

### 核心原则(prompt 重构)

MAIN_AGENT_PROMPT 的开头从「你是编排者」改为「你是**交互式编排者**:每一步都跟作者确认+给建议,不自己一口气跑完」。

铁律:**每轮只做一件事**(收集1-2项 / 建一个阶段 / 写一章),做完就停下回复作者,给建议 + 问下一步。**绝不一口气自主跑完多个阶段。**

### 立项(CONCEPT)— 分步收集,每次 1-2 项 + 建议

不再一股脑列 7 项 missing。每轮:
1. get_novel_info 看 missing。
2. 只挑 missing 里**最重要的 1-2 项**,给**基于已知信息的建议**:
   - 缺书名:"你的小说叫什么?建议从你的构想看可以叫《XXX》"
   - 缺题材:"什么类型?建议:东方玄幻/都市/科幻..."
   - 缺简介:"一两句话概括核心故事?建议:从你说的看,可以是「XXX」"
   - 缺核心冲突:"主角想要什么 vs 什么阻碍?建议:..."
   - 缺字数目标:"每章多少字?建议:网文一般 2000-4000"
   - 缺世界观:"世界观是什么?建议:从你的题材看..."
   - 缺文风:"什么文风?建议:冷峻/热血/轻松..."
3. 用户回答 → update_novel → 再问下一批 1-2 项。
4. missing 空 → 停下问:"基础信息齐了。建议下一步建参考资料。要开始吗?"

### 建置流水线 — 每步做完停下问

| 步骤 | 做完后的回复模板 |
|---|---|
| curator | "参考资料建好了(N 条)。建议下一步建世界观。要看参考再继续?" |
| worldbuilder | "世界观建好了(N 条目,score X)。建议下一步建大纲。要调整世界观吗?" |
| outliner | "大纲建好了(N 卷 M 弧 K 章细纲,score X)。建议下一步建角色档案。要看大纲吗?" |
| character | "角色档案建好了(N 个,score X)。建议开始写第 1 章。要调整角色吗?" |

**绝不自动从一步跑到下一步。**

### 写作(ACTIVE)— 每章写完停下问

"第 N 章写完了(X 字,score Y)。建议:写下一章 / 调整本章 / 改大纲。要继续吗?"

### MAIN_ROLE_REMINDER 同步

```
【职责提醒】你是交互式编排者。
- 每轮【只做一件事】(收集1-2项 / 建一个阶段 / 写一章),做完就停,问作者 + 给建议。
- 【绝不】一口气自主跑完多个阶段(curator→world→outline→character 一轮跑完 = 错)。
- 正文/设定/大纲/角色 一律 task 委派。
```

**不改**:DB / FE / AGENT_TREE / 工具 / 服务 / 代码逻辑。纯 prompt 行为变更。

## 显式不做(non-goals)

- **不加代码级「暂停/确认」门控**(如要求用户回复后才放行下步)——靠 prompt 让 main 每步停。代码级门控是 future(需 API 设计「确认」机制)。
- **不改 recursionLimit**(500 已合适——单步不会超;若 main 不听话跑多步,500 兜底截断)。
- **不改 settler/validator/writer 等子 agent**——它们是聚焦执行者,不需要交互式。
- **不改 FE**(chat UI 已支持多轮交互)。

## 测试

- 无新单测(prompt 文本改动,惯例不测)。
- 回归:`pnpm test` 不回归 + `pnpm typecheck` 过。
- **L2 实测验证**(最重要):改后跑 L2,看第 1 幕(立项)是否变成多步交互(而非一轮跑完)——L2 剧本可能需调整(立项幕拆成多步)。
- 活 E2E:新建小说,看 main 是否每步停下问。

## 验证未覆盖

- prompt 能否真让 deepseek「每步停」依赖模型——可能需要多轮调 prompt 才稳定。L2 实测是唯一验证手段。
